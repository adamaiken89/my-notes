# User Interface

- Think about the use cases
  - What input to be?
    - start with the raw input, assuming the user can tell you every time
    - refine the input control in the 2nd step
      - date picker
      - range selector
      - number input
      - email field
      - password field
  - What output to be? It could be dynamically according to the input / real time update
  - Never, ever start with the layout (solve the problem, fundamentally)
  - go for agility (start the real work as early as possible)
  - do not leave design UI to be WIP, simple UI is the best

- choose the color, font size, font typeface, border
- and define what they mean
- limit the choice of font size & color to be less than 10

Same common cases

1) Header
2) Content
3) Form Instruction
4) Form Input
5) Menu
6) Buttons

## Hierarchy

Apart from font size, font weight, font color, or even opacity can do the job better

- lighter color - de-emphasize

### White / Black color with opacity rather than grey text

<https://m2.material.io/design/color/text-legibility.html#text-backgrounds>

-> same HUE with different saturation and brightness

### Common Pattern

- Original Font color (usually black)
- Grey / Light Grey for content / sub-header
- Black but smaller and light weight for content

### Navigation

Tell users where are they out of the current options

- color highlight
- font weight
- different color
